修復代碼詳解

// 【重要修復】客戶端篩選：只顯示實際停靠在該車站的列車
const filteredTrains = trains.filter(train => {
   // 檢查列車是否真的停靠在此車站
   const hasStationStop = train.StationID === currentStationId;
  
   if (!hasStationStop) {
    console.warn(`⚠️ 列車 ${train.TrainNo} 被過濾...`);
   }
  
   return hasStationStop;
});

// 保存過濾後的列車資料
allTrains = filteredTrains;

關鍵修改點: 使用 filter() 方法確保 train.StationID 必須等於 currentStationId，才能通過篩選進入 allTrains 陣列

已知限制與注意事項

  • API 時間窗口: TDX LiveBoard API 仍會回應前後 30-60 分鐘內的列車。 此修復在客戶端進行二次驗證，確保只顯示真正停靠的列車。
  • StationID 可用性: 修復依賴 API 回應中的 StationID 欄位。 如果 API 不返回此欄位，篩選可能會失效。
  • 向下兼容: 修復完全兼容現有系統，不會影響已正確顯示的列車。
  • 性能: 篩選操作為簡單的 O(n) 循環，性能影響極小。
