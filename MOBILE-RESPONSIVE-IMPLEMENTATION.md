## 🔧 技術實施細節

### CSS 媒體查詢
```css
/* 手機優化 (≤768px) */
@media (max-width: 768px) { ... }

/* 超小螢幕 (≤480px) */
@media (max-width: 480px) { ... }

/* 平板 (600-959px) */
@media (min-width: 600px) and (max-width: 959px) { ... }

/* 橫屏 */
@media (max-height: 500px) and (orientation: landscape) { ... }

/* 觸控裝置 */
@media (hover: none) and (pointer: coarse) { ... }
```

### HTML 最佳實踐
```html
<!-- Viewport 設定 -->
<meta name="viewport" content="width=device-width, initial-scale=1.0">

<!-- 安全區域支援 -->
<meta name="viewport" content="viewport-fit=cover">

<!-- 主題顏色 -->
<meta name="theme-color" content="#667eea">
```

### CSS 單位系統
- 使用 `rem` 為主要單位
- 使用 `em` 為相對單位
- 避免硬編碼像素值
- 支援用戶縮放設定

## 🎓 開發團隊建議

### 編碼標準
1. **始終使用 rem 單位** 而非 px
2. **移動優先方法** - 先設計手機，再加大螢幕
3. **最小 44px 觸擊目標** 確保無障礙性
4. **始終測試** 至少三個斷點: 480px, 768px, 1200px+
5. **優化圖片** 用於行動裝置
6. **使用語義 HTML** 確保螢幕閱讀器相容

### 測試流程
```
1. 手動測試 (所有主要斷點)
   ├─ 480px (小手機)
   ├─ 768px (大手機)
   ├─ 1024px (平板)
   └─ 1920px (桌機)

2. 自動化測試
   ├─ Lighthouse 檢查
   ├─ WAVE 無障礙性檢查
   └─ Responsive Design Checker

3. 真實裝置測試
   ├─ iPhone
   ├─ Android
   └─ 平板
```
